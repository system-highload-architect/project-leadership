# Project Leadership — Архитектурные шаблоны и практики для Tech Lead / Architect

**Этот репозиторий — не просто набор кода. Это живая система знаний для инженеров, которые хотят строить устойчивые, масштабируемые и эволюционирующие системы.**

Здесь собраны архитектурные паттерны, стили организации кода, примеры реализации и документация, которые я использую и развиваю в своей практике как Tech Lead / Software Architect.

---

## 🧭 Навигация

Репозиторий построен по трём уровням абстракции:

1. **Архитектурные паттерны** — выбор стратегии для всей системы.
2. **Стили организации кода** — как структурировать код внутри системы.
3. **Примеры реализации паттернов** — готовые решения на Go для конкретных задач.

---

## 📂 Структура репозитория

### 🏛️ Уровень 1: Архитектурные паттерны (для всей системы)

📁 **[architecture-patterns/](architecture-patterns/README.md)** — выбор архитектуры для всего проекта.

| Раздел | Описание |
|--------|----------|
| `monolithic/` | Монолитная архитектура: модульный монолит, микрокернел |
| `service-oriented/` | SOA без XML, современные подходы с REST/gRPC |
| `microservices/` | Все варианты микросервисов: с API Gateway, CQRS, Event Sourcing, Saga, Sidecar |
| `event-driven/` | Событийно-ориентированные архитектуры: EDA, Event Sourcing, Actor Model |
| `distributed-systems/` | P2P, Leader-Follower, Sharding, Space-Based Architecture |
| `modern-hybrid/` | Serverless, Service Mesh, Broker Architecture |
| `anti-patterns/` | Что делать не стоит: Distributed Monolith и другие ловушки |

### 🔧 Уровень 3: Примеры реализации паттернов на Go

📁 **[patterns-examples/](patterns-examples/README.md)** — готовые примеры кода для повторного использования.

| Паттерн | Назначение |
|---------|------------|
| `circuit-breaker/` | Защита от каскадных отказов |
| `retry-backoff/` | Повторные попытки с экспоненциальной задержкой |
| `saga-choreography/` | Распределённые транзакции без центрального координатора |
| `saga-orchestration/` | Распределённые транзакции с центральным оркестратором |
| `event-sourcing/` | Хранение событий как источника истины |
| `cqrs/` | Разделение команд и запросов |
| `strangler/` | Постепенная замена частей системы |
| `sidecar/` | Вынос вспомогательных функций (логирование, метрики) |
| `ambassador/` | Прокси для внешних сервисов |
