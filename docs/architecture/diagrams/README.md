# 📊 Диаграммы архитектуры

**Эта директория содержит все визуальные представления архитектуры системы.**  
Диаграммы — это **основной инструмент для визуализации архитектуры**, позволяющий быстро понять структуру системы, взаимодействия между компонентами и потоки данных.

Все диаграммы хранятся в **текстовом формате PlantUML** и автоматически генерируются в SVG для встраивания в документацию. Это обеспечивает:
- Версионирование изменений вместе с кодом.
- Возможность редактирования в любом текстовом редакторе.
- Автоматическую генерацию актуальных картинок при каждом изменении.

## 🧠 Принципы построения диаграмм

1. **Одна диаграмма — одна цель.** Каждая диаграмма отвечает на конкретный вопрос (например, «как сервисы взаимодействуют при создании заказа?»).
2. **Единый инструмент — PlantUML.** Все диаграммы создаются в PlantUML для единообразия и автоматизации.
3. **Текстовый формат = версионирование.** Исходные `.puml` файлы хранятся в репозитории, а сгенерированные SVG — рядом.
4. **Автоматическая генерация.** При каждом изменении `.puml` файлов GitHub Actions автоматически перегенерирует SVG.
5. **Синхронизация с кодом.** Диаграммы должны отражать актуальное состояние системы. Изменение кода → изменение диаграмм.

## 🚀 Как работать с диаграммами

### Редактирование
1. Открой нужный `.puml` файл в VS Code с установленным расширением PlantUML.
2. Внеси изменения в код диаграммы.
3. Сохрани файл — предпросмотр обновится автоматически.
4. Запушь изменения в репозиторий.

### Генерация SVG (локально)
```bash
# Установка PlantUML (если ещё не установлен)
brew install plantuml   # macOS
sudo apt-get install plantuml   # Linux

# Генерация всех SVG
plantuml -tsvg docs/architecture/diagrams/**/*.puml
```

### Просмотр в документации
В Markdown-файлах ссылайся на сгенерированные SVG:
```markdown
![Контекстная диаграмма](diagrams/c4/context.svg)
```

## 📌 Важно

- **Не храни бинарные файлы** (PNG, JPEG). Используй только SVG или исходные `.puml`.
- **Проверяй генерацию SVG** после каждого изменения `.puml` (локально или через CI).
- **Синхронизируй с ADR** — если принято новое архитектурное решение, обнови соответствующие диаграммы.
