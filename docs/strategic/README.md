# 📈 Стратегические документы

**Эта директория содержит документы, определяющие стратегические цели, направления и ограничения проекта.**  
Они нужны для согласования видения между бизнесом, стейкхолдерами и командой разработки.

Здесь собраны ответы на ключевые вопросы:
- **Зачем мы это делаем?** (видение, цели, ценности)
- **Что и когда мы делаем?** (дорожная карта, этапы)
- **Как мы это делаем?** (архитектурные принципы, технологический стек)
- **Сколько это стоит?** (бюджет, ресурсы, люди)

## 🧠 Принципы стратегического планирования

1. **Видение — это компас.** Все решения, от архитектурных до продуктовых, должны соответствовать видению.
2. **Дорожная карта — это не догма.** Она должна быть гибкой и адаптироваться под изменения рынка и потребностей.
3. **Компромиссы — это норма.** Архитектура — это всегда баланс между скоростью, стоимостью, качеством и рисками.
4. **Документы — живые.** Они обновляются по мере развития проекта и появления новых данных.

## 📌 Как использовать стратегические документы

1. **Начни с `product-vision.md`** — чтобы понять, зачем мы вообще строим эту систему.
2. **Изучи `roadmap.md`** — чтобы знать, какие этапы нас ждут и в какие сроки.
3. **Ознакомься с `architecture-strategy.md`** — чтобы понять, какие технологические и архитектурные решения приняты и почему.
4. **Посмотри `budget-resources.md`** — чтобы оценить доступные ресурсы и ограничения.

Все стратегические документы должны быть **согласованы между собой** и служить основой для более детальных архитектурных и проектных решений (ADR, техническое задание, план спринта и т.д.).
