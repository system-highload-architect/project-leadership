# Бюджет и ресурсы: Robot Management System (RMS)

**Версия:** 1.0  
**Дата:** 2026-07-22  
**Статус:** Черновик

---

## 1. Введение

Данный документ описывает **бюджет и ресурсы**, необходимые для разработки, эксплуатации и поддержки системы на каждом этапе дорожной карты. Он помогает стейкхолдерам и менеджерам планировать затраты, распределять людей и оборудование, а также оценивать риски, связанные с финансированием.

Бюджет и ресурсы тесно связаны с этапами развития, описанными в [roadmap.md](roadmap.md), и архитектурными решениями из [architecture-strategy.md](architecture-strategy.md).

---

## 2. Команда проекта

### Основные роли

| Роль | Описание | Требуемый уровень |
|------|----------|-------------------|
| **Tech Lead / Архитектор** | Проектирование архитектуры, принятие решений, управление техническими рисками | Senior+ |
| **Backend-разработчики** | Реализация API, бизнес-логики, интеграций | Middle / Senior |
| **Frontend-разработчик** | Разработка Web UI для операторов и администраторов | Middle / Senior |
| **DevOps-инженер** | Инфраструктура, CI/CD, мониторинг, безопасность | Middle+ |
| **QA-инженер** | Тестирование, автоматизация тестов, обеспечение качества | Middle |
| **Product Manager** | Управление требованиями, коммуникация со стейкхолдерами | Middle+ |
| **Аналитик** | Сбор требований, моделирование данных, документация | Middle |

### Текущий состав (старт)

| Роль | Количество | Занятость |
|------|------------|-----------|
| Tech Lead / Архитектор | 1 | 100% (на старте) |
| Backend-разработчики | 2 | 100% |
| Frontend-разработчик | 1 | 100% |
| DevOps-инженер | 0.5 | 50% (разделяет с другими проектами) |
| QA-инженер | 0.5 | 50% (частичная занятость) |
| Product Manager | 0.5 | 50% |
| Аналитик | 0.2 | 20% (по мере необходимости) |

**Итого на старте:** ~4.7 FTE (Full-Time Equivalent).

## 5. Прогноз рисков и управление бюджетом

| Риск | Влияние | Меры смягчения |
|------|---------|----------------|
| **Перерасход средств** | Высокое | Ежеквартальный аудит затрат, пересмотр scope, оптимизация инфраструктуры |
| **Задержка найма** | Среднее | Гибкость в распределении задач, привлечение внештатных специалистов |
| **Рост нагрузки** | Среднее | Масштабирование по требованию (autoscaling), рефакторинг критических частей |
| **Изменение требований** | Среднее | Гибкий процесс, частые релизы, поддержка обратной совместимости |
