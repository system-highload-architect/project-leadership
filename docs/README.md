# 📚 Документация проекта

**Эта директория содержит всю архитектурную, проектную и операционную документацию.**  
Здесь собраны стратегические решения, технические спецификации, процессы работы с командой и инструкции для эксплуатации системы.

Документация построена так, чтобы быть **живой, актуальной и полезной** для всех участников проекта: от стейкхолдеров до разработчиков и DevOps-инженеров.

## 🧠 Принципы ведения документации

1. **Актуальность** — документы обновляются при изменении системы или процессов.
2. **Доступность** — вся документация находится в репозитории, доступна каждому члену команды.
3. **Структурированность** — каждый раздел имеет чёткое назначение и связан с другими.
4. **Прагматичность** — не перегружаем лишними деталями, пишем по делу.
5. **Единый формат** — все документы используют шаблоны из `templates/`.

## 🚀 Как использовать документацию

1. **Начни с `strategic/`**, чтобы понять цели и ограничения проекта.
2. **Изучи `architecture/`** для понимания технических решений и структуры системы.
3. **Ознакомься с `project-management/`** для планирования работы.
4. **Используй `people-processes/`** для организации команды и коммуникации.
5. **Следуй `operations/`** для развёртывания и поддержки.
6. **Бери шаблоны из `templates/`** для создания новых документов.

---

## 📌 Важно

- Все изменения в системе должны быть **отражены в документации** (особенно ADR и диаграммы).
- Документы пишутся на **русском языке** (для единообразия) с использованием терминов на английском, где это уместно.
- Для диаграмм используется **PlantUML** — они хранятся в текстовом виде и генерируются автоматически.
